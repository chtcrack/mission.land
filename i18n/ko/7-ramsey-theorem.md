# 7 — 램지 정리: Lean으로 형식화하라

## 문제

램지는 1930년에 증명했습니다: 임의의 색 수 r과 클리크 크기 k에 대해 N이 존재하여, {0, …, N−1} 위 완전 그래프의 간선을 어떤 방식으로 r-채색하더라도 단색 k-클리크가 반드시 포함된다. 이것은 램지 이론의 초석이 되는 정리입니다 — mission 1이 사냥하는 수들(R(5,5) 같은)이 존재하는 이유가 바로 이 정리죠. **하지만 이 유한 정리는 Lean/mathlib에서 한 번도 형식화된 적이 없습니다.** mathlib에는 가까운 이웃들 — Hales–Jewett, Hindman 정리 — 이 있지만, 유한 램지 정리 자체를 찾아보면 빈손으로 돌아옵니다.

형식화 공백을 메우는 일은 형식 수학의 세계에서 진짜 미해결 작업입니다. 완성하면 mathlib에 upstream하는 것도 고려해 보세요.

명제는 `challenge/Challenge.lean`에 고정되어 있습니다:

```lean
theorem ramsey (r k : ℕ) :
    ∃ N : ℕ, ∀ C : ℕ → ℕ → Fin r, ∃ (c : Fin r) (S : Finset ℕ),
      S.card = k ∧ (∀ n ∈ S, n < N) ∧
      ∀ i ∈ S, ∀ j ∈ S, i < j → C i j = c
```

(간선은 i < j인 쌍에서만 읽히므로 채색 함수는 대칭일 필요가 없습니다. 클리크 S는 {0, …, N−1} 안에 있어야 합니다. 이것은 다색·대칭 클리크 크기 형식이며, 비대칭 R(s,t) 버전은 k = max(s, t)로 두면 여기서 따라 나옵니다.)

| 정리 | 의미 | 점수 |
|---|---|---|
| `ramsey` | 유한 램지 정리의 형식화 | 1 |
| `ramsey_sanity` | r = 2, k = 2 사례 — 파이프라인 점검용 | 0 |

검증기([leanprover/comparator](https://github.com/leanprover/comparator))는 증명을 고정된 명제와 대조하고, 금지된 공리(`sorry`, 임의의 axiom, `native_decide`)를 거부하며, 모든 것을 Lean 커널로 재검사합니다.

## 점수

증명은 불리언입니다 — 이 고정된 정리를 증명하거나 못 하거나 둘 중 하나여서, 오를 순위는 없고 검증기가 `witness.theorems`에서 도출하는 "풀었는지" 플래그만 있습니다: 완전한 정리는 풀린 것으로 간주되고, sanity 사례로는 아닙니다. 이 미션은 **정복전**입니다: 최초로 승인된 증명이 현상금을 가져갑니다. mission 6과 마찬가지로 (mission 5와 달리) 문샷이 아닙니다 — 수학은 백 년 전 것이고 교과서 증명은 짧습니다. 남은 것은 꼼꼼한 증명 엔지니어링입니다.

이 플래그를 직접 정할 필요는 없습니다 — record에서 `score`를 생략하면 검증기가 계산합니다(기재한 경우에만 그 값을 대조합니다).

## Witness 형식

```json
{
  "mission": "7-ramsey-theorem",
  "author": "your-handle",
  "date": "YYYY-MM-DD",
  "witness": {
    "theorems": ["ramsey"],
    "solution": "import Mathlib\n\ntheorem ramsey ... := by\n  ..."
  }
}
```

- `witness.theorems`에는 solution이 증명한 고정 정리의 이름을 적습니다. 이것이 진짜 신고이며, "풀었는지" 플래그는 여기서 도출됩니다.
- `witness.solution`은 `Solution.lean`의 전문입니다.
- `score`는 선택 사항이며 도출됩니다 — 생략하거나(위와 같이), 포함한다면 선언한 정리들 중 최고 점수와 같아야 합니다.
- 표준 공리만 허용됩니다. mathlib(고정 버전)과 보조 보조정리는 자유입니다.

검증: `python3 verify.py <record.json>`

필요한 환경: [elan](https://leanprover-community.github.io/get_started.html). 첫 실행에는 네트워크가 필요합니다 (고정 툴체인, mathlib 캐시, 한 번의 comparator 빌드. 이후 `~/.cache/mission-land/`에 캐시).

## 문헌 기록

정리 자체: F. P. Ramsey, *On a problem of formal logic* (1930). Isabelle, Mizar 등에는 형식화가 있습니다. Lean에는 인접한 외부 작업(Bhavik Mehta의 대각 램지 개발 등)이 있지만 mathlib에 이 정리는 없습니다. 여기서 최초로 승인된 Lean 형식 증명이 기록을 세웁니다.

## 알려진 접근법

- **경로 1 — 이중 귀납법 (권장)**: 교과서 증명입니다. 두 색의 경우 s + t에 대한 귀납으로 R(s, t) ≤ R(s−1, t) + R(s, t−1)을 사용합니다: 꼭짓점 하나를 골라 그 간선들을 색으로 비둘기집 분류하고, 다수파 이웃으로 재귀합니다. 색 수 r로의 일반화는 r에 대한 귀납(두 색을 합치거나, r-갈래 비둘기집으로 같은 논증을 반복). 종이 위에서는 짧지만, Lean에서는 이웃 제한을 깔끔하게 다루는 것이 관건입니다 — 귀납은 ℕ의 접두사가 아니라 "임의의 N-원소 인덱스 집합의 채색" 위에 세우세요.
- **경로 2 — 무한 버전 먼저**: 같은 비둘기집 논증(장부 정리 불필요)으로 무한 램지 정리(ℕ의 쌍에 대한 임의의 유한 채색은 무한 단색 집합을 가진다)를 증명한 뒤, 콤팩트성/König 보조정리로 유한 형식을 추출합니다. 도구는 늘지만 각 절반은 더 깔끔합니다.
- **경로 3 — 이식**: mathlib 밖에는 이미 Lean으로 된 램지 상계 개발이 있습니다. 여기 고정된 정확한 명제에 맞게 옮겨 쓸 수 있습니다. 라이선스에 주의하세요.
- 퇴화 매개변수(r = 0, k ≤ 1)는 자명하게 성립하도록 쓰여 있습니다 — 일반 논증이 이를 커버하는지, 아니면 따로 처리할지 확인하세요.
